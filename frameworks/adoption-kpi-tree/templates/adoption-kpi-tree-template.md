```text

```

A stronger ambition describes the intended change in work.

```text
Enable [target group] to use [capability] in [workflow] so that [value] improves while [quality/risk/accountability condition] remains clear.
```

## 3. Adoption outcomes

Adoption outcomes describe what should become true if adoption works.

They should be more concrete than the strategic ambition, but still outcome-oriented.

Examples:

* relevant workflows include the new capability where useful
* teams know when and how to use it appropriately
* human judgment points are clear
* review effort becomes more focused
* outputs become more consistent
* risky use cases are escalated earlier
* successful patterns are reused across teams

## 4. Behavioral and workflow signals

Behavioral and workflow signals show whether adoption is appearing in everyday work.

Ask:

```text
What would we need to see in the work system to know that adoption is actually happening?
```

Examples:

* first meaningful use completed
* recurring workflows use the new method
* approved templates are used in real cases
* old workarounds decrease
* teams use the intended review path
* questions become more specific over time
* recurring issues are fed back into templates or guidance

* fewer avoidable review loops
* better audience or use-case fit

Examples:

* missing review steps
* unclear accountability
* sensitive use cases without escalation
* unmanaged tool use
* repeated policy confusion
* overreliance on generated output

## 9. Countermetrics

Countermetrics help prevent one-sided adoption measurement.

For every growth or value metric, ask what could go wrong if this number improves too quickly or without enough context.

Examples:

| Growth or value metric   | Countermetric                       |
| ------------------------ | ----------------------------------- |
| More AI-supported drafts | Higher review rejection rate        |
| Faster cycle time        | More downstream corrections         |
| More self-service        | More unresolved edge cases          |
| More automation          | More unclear accountability         |
| More reuse               | More generic or low-context outputs |
