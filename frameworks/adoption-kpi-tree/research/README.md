# Research

This folder maps the research fields, professional frameworks, expert resources and case references behind the Adoption KPI Tree.

The Adoption KPI Tree is a practical framework.

It is not a scientific literature review.

However, it is informed by several established bodies of knowledge:

* change management
* technology acceptance
* product and UX measurement
* software delivery and productivity measurement
* AI governance
* enterprise AI transformation
* organizational learning

## Purpose

The purpose of this folder is to show where the Adoption KPI Tree connects to existing research and expert practice.

It helps answer:

```text
Which established frameworks inform this approach?
```

```text
Which expert resources are useful for deeper learning?
```

```text
Where does the Adoption KPI Tree add practical operating structure?
```

## Core positioning

Most adoption measurement systems over-measure activity and under-measure operating change.

They often track:

* training participation
* tool access
* communication reach
* login counts
* active users
* pilot count
* number of generated outputs

These indicators can be useful.

But they do not prove that adoption has changed work, improved quality, created value, clarified accountability or reduced unmanaged risk.

The Adoption KPI Tree asks:

```text
What would we need to see in the work system to know that adoption is actually happening?
```

## How this research layer supports the framework

The Adoption KPI Tree connects:

```text
Strategic ambition
→ adoption outcomes
→ behavioral and workflow signals
→ quality, value and risk indicators
→ decision logic
→ operating action
```

The research layer explains why those elements matter.

For example:

| Research area                 | Contribution to the Adoption KPI Tree                            |
| ----------------------------- | ---------------------------------------------------------------- |
| Change management             | Adoption speed, utilization, proficiency and reinforcement       |
| Technology acceptance         | Usefulness, ease of use, intention and actual use                |
| Product and UX measurement    | Goals, signals, metrics and task success                         |
| Software delivery measurement | Speed, stability, productivity dimensions and countermetrics     |
| AI governance                 | Risk, accountability, lifecycle oversight and human review       |
| Enterprise AI transformation  | Work redesign, role redesign, maturity and scaling               |
| Organizational learning       | Interpretation, feedback loops, escalation and operating rituals |

## What this folder is not

This folder is not:

* a complete academic literature review
* a ranking of all adoption frameworks
* legal, compliance or certification guidance
* a substitute for domain-specific expert review
* a claim that one framework covers all adoption contexts

It is a practical research map for adoption measurement and operating-system design.

## Final principle

The Adoption KPI Tree is strongest when it combines research credibility with operational usefulness.

It is not only interested in whether people use a tool.

It is interested in whether work changes, value becomes visible, risk becomes governable and signals trigger better decisions.
